Architecture

The core challenge was bridging the synchronous nature of typical web interfaces with the asynchronous, compute-heavy requirements of medical image analysis. We designed an event-driven architecture utilizing GraphQL subscriptions.

GraphQL & YOLOv8 Integration Pipeline

  1. Client uploads panoramic radiograph via GraphQL mutation.
  2. Backend API queues the image to a specialized GPU-backed worker pool.
  3. YOLOv8 inference runs, detecting caries, bone loss, and structural anomalies.
  4. Results (bounding boxes, confidence scores) are structured into a standardized medical JSON payload.
  5. GraphQL subscription pushes the completed analysis back to the active Vue 3 client.
query GetAnalysis($id: ID!) {
  radiographAnalysis(id: $id) {
    status
    findings {
      type       // e.g., "CARIES"
      confidence // 0.98
      boundingBox { x, y, width, height }
    }
  }
}

Challenges

Asynchronous State Management

Handling the indeterminate wait time for YOLOv8 inference required a robust state machine in Vue. We implemented a polling fallback for the GraphQL subscriptions to ensure UI consistency if WebSockets dropped during heavy inference loads.

Medical Prediction Rendering

Drawing precise, non-destructive bounding boxes over high-resolution DICOM/JPEG radiographs in the browser was computationally expensive.

Solution: We utilized HTML5 Canvas overlaid perfectly on the image via absolute positioning, rendering the YOLOv8 coordinate data dynamically. This allowed toggling different anomaly layers (e.g., hiding 'Bone Loss' to focus on 'Caries') without re-rendering the heavy base image.
